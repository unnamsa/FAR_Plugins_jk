Версия 2.0.0
------------
1. Появилась возможность подключать к плагину LUA-программы для просмотра внутренней структуры
   элементов данных HIVE-файла (команда hive:explore).

2. Появилась возможность экспорта ключей и значений HIVE-файла в .reg-файл (команда hive:export).

3. Появились LUA-команды Parse, GetKeyItem и GetValItem.

4. В командах cd, GetItem, GetKeyItem и GetValItem  появилась опция "-nocase".

5. Переработана функция копирования (F5) элементов данных HIVE-файла в реальную файловую систему.
   В частности, копирование может быть в любой момент прервано нажатием на Ctrl-Break.

6. Поддержано сохранение просматриваемых виртуальных директорий HIVE-файла в истории
   просмотренных директорий FAR-а (если задан соответствующий параметр конфигурации плагина), 
   а также запуск плагина из истории директорий FAR-а (Alt-F12).

7. Существенно ускорен первичный разбор HIVE-файла при его открытии.

8. Алгоритм разбора Security Descriptor приведен в соответствие с [MS-DTYP]
   (декомпиляция условий в условных ACE пока не реализована).

9. Корректно обрабатывается область HIVE-файла, расположенная за последним HBIN-ом.

   


Версия 1.0.0
------------

Основные отличия от RegEdit-а :

1. Позволяет с точностью "до байта" исследовать содержимое файлов системного реестра.
   Для этого предусмотрен режим просмотра (RawView) на уровне базовых структур (HBIN и CELL) HIVE-файла.

2. Позволяет оперировать средствами FAR-а со списками ячеек определенного типа.
   Это, к примеру, дает возможность отсортировать список ключей по времени модификации
   или список ячеек данных в порядке убывания их размеров...

3. Позволяет в один клик переключаться между TreeView и RawView.
   Таким образом, "стоя" на любом ключе можно посмотреть, как он выглядит изнутри
   и в каком месте HIVE-файла находится соответствующий CELL.

4. Позволяет исследовать содержимое HIVE-файла, недоступное для стандартных средств.
   Например, можно изучать содержимое "пустых" ячеек (на предмет поиска чего-либо недавно удаленного :-)
   или неиспользуемые "хвосты" reference list-ов...

5. Позволяет в один клик переключаться в режим шестнадцатеричного отображения
   любого элемента данных или HIVE-файла в целом. Разумеется, в этом режиме
   есть возможность поиска шестнадцатеричного контекста ;-)

6. Позволяет работать с HIVE-файлом из LUA-программ.

7. Позволяет вызывать LUA-программы в контексте открытого HIVE-файла.
   Вызываемые программы размещаются в поддиректории Artifacts директории плагина.

8. НЕ позволяет редактировать HIVE-файл :-)))
